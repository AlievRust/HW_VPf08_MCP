# Proposal: notes-mcp учебный проект

## Problem
В репозитории нет рабочего кода, хотя по исходной идее нужен полностью готовый учебный проект с MCP-сервером заметок и CLI-клиентом, который умеет работать с этим сервером через tools.

## Goal
С нуля создать минимальный, но полноценный проект `notes-mcp`:
- MCP-сервер заметок на SQLite;
- CLI-клиент с интерактивным чатом;
- интеграцию с OpenAI для tool calling;
- конфигурацию через `.env`;
- документацию для запуска и проверки.

## Scope
Входит:
- хранение заметок в локальной SQLite базе;
- автосоздание и первичное заполнение базы тестовыми заметками;
- MCP tools для списка, чтения, создания, поиска, обновления и удаления заметок;
- CLI-оболочка, которая передает запросы пользователя в LLM и выполняет tool calls;
- понятные сообщения об ошибках и debug-режим;
- README, `.env.example`, `.gitignore` и базовые проектные документы.

## Out of scope
Не входит:
- веб-интерфейс;
- удаленная база данных;
- аутентификация и роли;
- многопользовательский режим;
- синхронизация между устройствами;
- произвольные REST API поверх MCP;
- сложные расширяемые плагины.

## User-visible behavior
Пользователь сможет:
- запустить сервер командой `python server.py`;
- запустить CLI командой `python cli.py`;
- написать обычным языком запрос вроде «покажи все заметки» или «создай заметку про MCP»;
- получить человеческий ответ без сырых JSON-структур;
- включить debug-режим и увидеть детали tool calls и ответов сервера;
- выполнять операции над заметками через MCP tools.

## Risks
- Неверная настройка MCP-транспорта может сломать связку CLI и сервера.
- Неполная валидация пользовательского ввода может привести к плохому UX.
- Ошибки интеграции с OpenAI могут проявляться как некорректные tool calls.
- Первичное заполнение базы должно быть идемпотентным, иначе появятся дубликаты.

## Acceptance criteria
- Сервер запускается локально и автоматически создает `notes.db`.
- При первом старте в базе есть 20 тестовых заметок.
- Доступны MCP tools: `list_notes`, `get_note`, `create_note`, `search_notes`, `update_note`, `delete_note`.
- CLI читает параметры из `.env`, подключается к серверу и использует OpenAI модель из конфигурации.
- CLI корректно обрабатывает `/exit`, `/help`, `/tools`, `/debug on`, `/debug off`.
- Ошибки сервера и неправильные tool calls не приводят к падению CLI.
- Документация описывает установку, запуск и проверку сценариев.
