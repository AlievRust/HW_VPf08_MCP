# Design: настраиваемое окно истории диалога

## Текущее поведение
CLI отправляет в модель только текущий пользовательский запрос. История диалога хранится в памяти, но не используется как контекст следующего запроса.

## Целевое поведение
CLI хранит последние `N` сообщений диалога и использует их как контекст для следующего запроса.

## Правила
- Значение `N` берется из `.env` через `CHAT_HISTORY_LIMIT`.
- Если переменная не задана, используется `10`.
- Значение должно быть целым числом больше нуля.
- В историю попадают только сообщения `user` и `assistant`.
- Системное сообщение не входит в окно истории.
- Перед отправкой нового запроса модель видит не более `N` последних сообщений диалога.

## Технический подход
- `cli_client/config.py`:
  - добавить поле `chat_history_limit` в `AppConfig`;
  - загрузить `CHAT_HISTORY_LIMIT` из `.env`;
  - валидировать значение.
- `cli_client/cli.py`:
  - хранить сообщения диалога в памяти;
  - передавать в LLM-клиент только срез последних сообщений;
  - после ответа добавлять новую пару сообщений `user` + `assistant`;
  - обрезать историю до лимита.
- `cli_client/llm_client.py`:
  - принять историю сообщений как часть входного контекста;
  - сохранить текущую логику tool calling без изменений.

## Проверка
- unit-тест на загрузку `CHAT_HISTORY_LIMIT`;
- unit-тест на обрезку истории до заданного лимита;
- `python -m unittest discover -s tests`;
- `python -m compileall cli_client tests`.
