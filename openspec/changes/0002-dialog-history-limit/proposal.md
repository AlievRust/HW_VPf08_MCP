# Proposal: настраиваемое окно истории диалога

## Цель
Добавить в CLI-клиент хранение контекста последних сообщений диалога, чтобы модель учитывала предыдущие реплики пользователя и ассистента при следующих ответах.

## Пользовательский эффект
- CLI перестает отвечать каждый раз как на новый запрос.
- В следующих сообщениях модель видит последние сообщения диалога и может продолжать контекст.
- Размер окна истории настраивается через `.env`.

## Объем
- Вводится параметр окружения `CHAT_HISTORY_LIMIT`.
- По умолчанию используется значение `10`.
- В контексте учитываются только сообщения пользователя и ассистента.
- Системное сообщение и tool calls не считаются частью окна истории.

## Не входит в объем
- Постоянное хранение истории между запусками CLI.
- Сжатие/саммари длинной истории.
- Изменение MCP-сервера или схемы заметок.

## Риски
- Слишком большое окно истории может увеличить стоимость и задержку запросов.
- Слишком маленькое окно может привести к потере контекста в длинном диалоге.
